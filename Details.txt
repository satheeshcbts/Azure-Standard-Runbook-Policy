Azure Policy CANNOT directly enforce PIM (Privileged Identity Management).
Azure Policy CAN:
   1. Audit or deny RBAC role assignments (Owner, User Access Admin, RBAC Admin)
   2. Prevent permanent Owner assignments
   3. Prevent PIM circumvention (direct assignments instead of eligible)
   4. Enforce that only approved principals can assign Owner

What Azure Policy CANNOT do: Force a role to be “eligible‑only”, Require activation via PIM, Control PIM activation settings (MFA, approval, duration)

PIM enforcement is done via Entra ID PIM configuration, not policy.
Correct solution = Azure Policy + PIM working together.

Real‑world architecture (recommended by Microsoft)
Azure Policy
  ├─ Audit/Deny Owner & privileged role assignments
  └─ Prevent non‑PIM permanent assignments

Microsoft Entra PIM
  ├─ Roles assigned as Eligible
  ├─ MFA required on activation
  ├─ Time‑bound JIT access

This is the industry‑standard, Microsoft‑recommended model.
